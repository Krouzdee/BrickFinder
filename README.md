# BrickFinder

**BrickFinder** — это десктопное приложение с графическим интерфейсом для поиска и идентификации деталей конструктора Lego в реальном времени. 

Приложение позволяет пользователю создать собственную базу деталей, просто сфотографировав их или загрузив изображение. Затем, через веб-камеру или на статичном изображении, BrickFinder может находить эти детали, отображая рамку с названием и процентом уверенности.

**Ключевые возможности:**
- 🧱 **Добавление новых деталей:** Интуитивный интерфейс для пополнения базы деталей.
- 🔍 **Поиск в реальном времени:** Анализ видеопотока с веб-камеры.
- 🖼️ **Поиск на изображении:** Возможность загрузить статичное фото для анализа.
- 🎚️ **Настраиваемый порог уверенности:** Контроль точности распознавания через слайдер (0-100%).

## Требования к системе

- **Операционная система:** Windows 10/11 или Linux.
- **Python:** версии 3.8 или выше.
- **Аппаратное обеспечение:**
    - Веб-камера (для режима реального времени).
    - **Рекомендуется:** Видеокарта с поддержкой CUDA (NVIDIA) для ускорения нейросетей.
- **Память:** минимум 4 ГБ ОЗУ.

## Установка и запуск

Следуйте этой инструкции для запуска приложения на системе.

### 1. Клонирование репозитория

Скачайте исходный код проекта:

```bash
git clone https://github.com/Krouzdee/BrickFinder.git
cd BrickFinder
```

### 2. Создание виртуального окружения (рекомендуется)

Изолируйте зависимости проекта:

**Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

#### CPU (по умолчанию)
Если у вас нет видеокарты NVIDIA или вы не планируете использовать GPU, установите стандартные зависимости:
```bash
pip install -r requirements.txt
```

#### NVIDIA CUDA
Для работы приложения с использованием видеокарты NVIDIA необходимо установить CUDA Toolkit.

1.  Перейдите на официальный сайт: [https://developer.nvidia.com/cuda-downloads](https://developer.nvidia.com/cuda-downloads)
2.  Выберите свою операционную систему Windows/Linux.
3.  Скачайте и установите инсталлятор.
4.  После установки убедитесь, что переменные среды настроены (обычно это происходит автоматически). Проверить установку можно командой:
    ```bash
    nvcc --version
    ```

Установка PyTorch с поддержкой CUDA, соответствующей вашей версии драйвера.

1.  Определите версию CUDA, установленную на вашей системе (например, 11.8, 12.1). Это можно сделать командой `nvidia-smi` в терминале.
2.  Перейдите на официальный сайт PyTorch: [https://pytorch.org/get-started/locally/](https://pytorch.org/get-started/locally/)
3.  Выберите вашу версию CUDA в калькуляторе на сайте и выполните сгенерированную команду. Обычно это выглядит так (пример для CUDA 12.1):
    ```bash
    pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
    ```

После установки PyTorch с GPU, установите остальные библиотеки из `requirements-gpu.txt`:

```bash
pip install -r requirements-gpu.txt
```

### 4. Запуск приложения

Запустите главный файл:

```bash
python main.py
```

## Алгоритм работы

Приложение BrickFinder предназначено для распознавания и отслеживания деталей Lego в режиме реального времени. Его работа строится на комбинации предобученной модели компьютерного зрения YOLO для обнаружения объектов и пользовательских алгоритмов для идентификации конкретных деталей из базы данных.

### 1. Добавление новой детали в базу
1.  Пользователь нажимает кнопку **"Добавить деталь"**.
2.  В открывшемся окне вводится название детали и загружается изображение (с камеры или из файла).
3.  Система обрабатывает изображение:
    *   Изображение может быть уменьшено для повышения скорости обработки.
    *   Модель YOLO пытается обнаружить деталь на изображении. Если деталь найдена, её контур вырезается. Если нет, используется всё загруженное изображение.
    *   С вырезанного изображения удаляется фон (обрезаются 10% пикселей с каждой стороны).
    *   Из обработанного изображения извлекаются **признаки формы** (соотношение сторон) и **гистограмма цвета** (в цветовом пространстве RGB, бины 6×6×6).
    *   Оригинальное изображение (без фона) и извлеченные признаки сохраняются в папке `data` в виде `.jpg` и `.pkl` файлов, где имя файла — это безопасное ("очищенное") название детали.

### 2. Выбор цели для поиска
1.  В галерее на правой панели отображаются все сохраненные детали.
2.  Пользователь нажимает зеленую кнопку (`✔`) на карточке нужной детали.
3.  Система загружает из базы данных признаки (вектор формы и гистограмму цвета) выбранной детали.
4.  Текущая цель отображается в соответствующем поле интерфейса, и все последующие кадры будут анализироваться на наличие именно этой детали.

### 3. Обработка видеопотока или изображения
Это основной цикл распознавания, который запускается либо для кадров с камеры, либо для загруженного статичного изображения.

#### 3.1. Обнаружение объектов
*   Каждый кадр сначала передается в модель **YOLO** (`lego_detector200.pt`).
*   Модель возвращает ограничивающие рамки (bounding boxes) вокруг всех потенциальных деталей Lego, которые она умеет распознавать. Это позволяет отфильтровать области, где вообще нет подходящих объектов, и значительно ускоряет дальнейшую работу.

#### 3.2. Сравнение с эталоном (для выбранной цели)
Для каждого обнаруженного YOLO объекта выполняется следующий алгоритм:
1.  Из кадра вырезается область, ограниченная рамкой.
2.  С вырезанного изображения удаляется фон (те же 10% с каждой стороны).
3.  Из обработанного изображения извлекаются **гистограмма цвета** и **соотношение сторон** (вектор формы), аналогично тому, как это делалось при добавлении детали.
4.  Рассчитываются метрики схожести:
    *   **Схожесть формы (shape_sim)**: вычисляется как отношение меньшего соотношения сторон к большему. Значение близкое к 1 означает идентичную форму.
    *   **Схожесть цвета (color_sim)**: вычисляется с помощью метрики `cv2.HISTCMP_CORREL`. Значение преобразуется в коэффициент схожести от 0 до 1.
5.  **Итоговая уверенность (final_sim)** рассчитывается как среднее арифметическое схожести формы и цвета:
    `final_sim = (shape_sim + color_sim) / 2`
6.  В консоль выводится детальный лог для отладки: `YOLO: X.XXX | Shape: X.XXX | Color: X.XXX => TOTAL: X.XXX`.

#### 3.3. Визуализация результата
*   Если итоговая уверенность (`final_sim`) превышает заданный пользователем порог (по умолчанию 70%):
    *   Вокруг объекта рисуется прямоугольник.
    *   Цвет прямоугольника интерполируется от красного (низкая уверенность) до зеленого (высокая уверенность).
    *   Рядом с прямоугольником выводится название детали и процент уверенности.
    *   Для отрисовки текста с поддержкой кириллицы используется библиотека Pillow (изображение конвертируется в RGBA, текст и фон рисуются на прозрачном слое, который затем накладывается на исходный кадр).

### 4. Оптимизации производительности
Для работы в реальном времени используются следующие механизмы:
*   **Масштабирование кадра**: Если разрешение кадра слишком высокое, оно динамически уменьшается перед передачей в YOLO, а координаты боксов затем обратно масштабируются.
*   **Кэширование признаков**: Для каждого отслеживаемого YOLO объекта (по `track_id`) векторы формы и цвета кэшируются. Пересчет признаков для одного и того же объекта выполняется не каждый кадр, а с интервалом (`recompute_interval` = 5 кадров), что снижает вычислительную нагрузку.
*   **Пакетная обработка**: Признаки для нескольких новых объектов на кадре вычисляются группами (`batch_get_vectors`), что эффективнее, чем обработка в цикле.

### 5. Управление базой данных
*   **Удаление детали**: При нажатии красной кнопки (`❌`) на карточке, файлы `.jpg` и `.pkl` удаляются из папки `data`. Если удаленная деталь была текущей целью, поиск сбрасывается.
*   **Сброс цели**: Позволяет прекратить поиск конкретной детали, не удаляя её из базы. Детектор продолжит показывать кадры без выделений.

### 6. Работа со статическим изображением
Алгоритм для статического изображения полностью повторяет алгоритм обработки одного кадра видеопотока. После загрузки или смены цели, обработка запускается заново, и результат отображается на вкладке "Изображение".
