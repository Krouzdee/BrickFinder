# BrickFinder

**BrickFinder** — это десктопное приложение с графическим интерфейсом для поиска и идентификации деталей конструктора Lego в реальном времени. 

Приложение позволяет пользователю создать собственную базу деталей, просто сфотографировав их или загрузив изображение. Затем, через веб-камеру или на статичном изображении, BrickFinder может находить эти детали, отображая рамку с названием и процентом уверенности.

**Ключевые возможности:**
- 🧱 **Добавление новых деталей:** Интуитивный интерфейс для пополнения базы деталей.
- 🔍 **Поиск в реальном времени:** Анализ видеопотока с веб-камеры.
- 🖼️ **Поиск на изображении:** Возможность загрузить статичное фото для анализа.
- 🎚️ **Настраиваемый порог уверенности:** Контроль точности распознавания через слайдер (0-100%).

## Требования к системе

- **Операционная система:** Windows 10/11 или Linux.
- **Python:** версии 3.8 или выше.
- **Аппаратное обеспечение:**
    - Веб-камера (для режима реального времени).
    - **Рекомендуется:** Видеокарта с поддержкой CUDA (NVIDIA) для ускорения нейросетей.
- **Память:** минимум 4 ГБ ОЗУ.

## Установка и запуск

Следуйте этой инструкции для запуска приложения на системе.

### 1. Клонирование репозитория

Скачайте исходный код проекта:

```bash
git clone https://github.com/Krouzdee/BrickFinder.git
cd BrickFinder
```

### 2. Создание виртуального окружения (рекомендуется)

Изолируйте зависимости проекта:

**Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

**Linux / macOS:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

Установите все необходимые пакеты из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```


### 4. Запуск приложения

Запустите главный файл:

```bash
python main.py
```

## Использование

### Главное окно
Интерфейс состоит из двух панелей:
- **Левая панель (вкладки):** Здесь отображается видеопоток с камеры или загруженное изображение.
- **Правая панель (управление):** Содержит список деталей, настройки распознавания и кнопки управления.

### Добавление новой детали
1. Нажмите кнопку **«Добавить деталь»**.
2. В открывшемся окне введите **название детали**.
3. Загрузите изображение одним из способов:
    - Нажмите **«Сделать снимок с камеры»**, чтобы использовать текущий кадр.
    - Нажмите **«Загрузить фото»**, чтобы выбрать файл на диске.
4. Убедитесь, что деталь хорошо видна на предпросмотре, и нажмите **«Сохранить деталь»**. Деталь появится в галерее справа.

### Поиск детали
1. В галерее справа нажмите **зеленую кнопку** на карточке детали, которую хотите найти.
2. Убедитесь, что на вкладке **«Камера»** выбрано нужное устройство.
3. Нажмите кнопку **«Старт»**.
4. Настраивайте **порог уверенности** слайдером. Детали с уверенностью ниже порога не будут обведены рамкой.
5. Для поиска на статичном изображении переключитесь на вкладку **«Изображение»** и нажмите **«Выбрать изображение»**.

## Демонстрационный набор данных

Для проверки работоспособности системы предоставляется демонстрационный набор из 5 типов деталей в различных цветах (всего более 10 объектов).

## Технические детали

### Алгоритм работы

1.  **Детекция (Модель компьютерного зрения YOLO):** На каждом кадре обученная модель YOLO находит области, потенциально содержащие детали Lego.
2.  **Извлечение признаков (ConvNeXt-Tiny):** Для каждой обнаруженной области и для эталонного изображения из базы извлекаются векторы признаков (эмбеддинги) с помощью нейросети ConvNeXt-Tiny, предобученной на ImageNet.
3.  **Цветовая характеристика:** Дополнительно из области выделяется гистограмма цвета на основе доминантного цвета, что повышает устойчивость к изменению формы.
4.  **Сравнение (Метрика косинусного расстояния):** Вектор признаков найденной детали сравнивается с вектором эталона. Чем меньше косинусное расстояние, тем выше `shape_sim`.
5.  **Итоговая уверенность:** Финальный `confidence score` вычисляется как среднее арифметическое между оценкой формы (`shape_sim`) и оценкой цвета (`color_sim`).
6.  **Фильтрация:** Результаты, итоговая уверенность которых ниже заданного пользователем порога, отбрасываются.
